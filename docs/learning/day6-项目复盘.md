## 4. 面试必考：5 个技术难点

### 难点 1：SIGPIPE 崩溃的排查与修复

**问题**：音乐播放结束后，整个 `car_system` 进程直接退出，无 crash 日志。

**排查**：加 PID 日志追踪 → 发现 EOF 标志正确设置，但主线程来不及处理进程就退出了 → 怀疑信号 → 查 `man 7 signal` → SIGPIPE 默认行为是终止进程。

**根因**：mplayer 退出后 pipe 写端仍然打开，Writer 线程 `write()` 触发 SIGPIPE。

**修复**：`main()` 第一行 `signal(SIGPIPE, SIG_IGN)` 忽略该信号。

### 难点 2：Seek 拖进度条偶发卡死的竞态条件

**问题**：拖动进度条后进度停止更新（Writer 线程阻塞），再次拖动音乐卡死。

**根因**：旧 seek 流程使用 `pthread_kill` 链式触发信号处理 → `cond_wait` → `cond_signal`。存在"信号在 cond_wait 之前到达"的窗口期 → Writer 永久阻塞。

**修复**：
1. Seek 改用 `seek <val> 1`（绝对百分比），不暂停任何线程
2. Pause 改用标志位 `g_music_io_paused` + `sigaction`（无 SA_RESTART），主线程统一控制

### 难点 3：视频播放时 LVGL UI 闪烁

**问题**：视频播放时仪表盘和状态栏闪烁。

**根因**：mplayer 直接写 `/dev/fb0`（视频渲染），LVGL 的 `fbdev_flush` 也在写 `/dev/fb0`（UI 渲染）。两者争抢同一块显存。

**修复**：全局标志 `g_video_overlay_active`。所有 UI 刷新定时器在视频播放时 `return`，不执行 LVGL 刷新。视频关闭后恢复。

### 难点 4：跨线程更新 LVGL UI 的正确方法

**问题**：网络线程收到天气数据后需要更新状态栏标签。

**错误做法**：在网络线程直接调 `lv_label_set_text()` → 非线程安全，可能 crash。

**正确做法**：`lv_async_call(ui_refresh_status_labels, NULL)` — 把回调放入 LVGL 内部队列，由下一次 `lv_timer_handler()` 在主线程安全执行。

### 难点 5：WSL2 网络隔离问题的解决

**问题**：天气服务器在 WSL2 中运行，开发板无法连接。

**根因**：WSL2 有独立虚拟网卡（IP 如 `172.29.x.x`），与开发板不在同一子网。每重启 IP 都变。

**尝试过的方案**：
1. `netsh interface portproxy` 端口转发 — 可用但重启后规则丢失
2. 加强防火墙规则 — 部分解决但不能根治

**最终方案**：将天气服务器从 WSL2 迁移到 Windows 直接运行。WSL2 仅用于编译（`make`）。

## 5. 简历项目描述要点

**项目名称**：智能车机系统（ARM Linux + LVGL）

**个人职责**（根据你实际做的调整）：
- 实现多线程音乐/视频播放器（fork+pipe 架构，3 线程并发）

- POSIX 多线程并发（7 线程）

- TCP 网络编程（断线重连、缓冲拆包）
- LVGL GUI 框架（20+ 控件、事件驱动）
- 中文位图字体生成
- JSON 数据序列化/反序列化
