🚗 智能车机项目 – 从零开始完整步骤
本项目基于 GEC6818 开发板（分辨率800*480） + LVGL + 系统编程 + 网络编程，分 8个阶段 实现。每个阶段完成一个独立模块，可单独测试，最后集成。
📌 技术栈总览
C语言、Linux系统编程（多线程、IPC、文件IO）
网络编程（TCP Socket、HTTP、JSON）
LVGL 8/9（按钮、标签、容器、动画）
mplayer（音视频播放）
🔰 第一阶段：基础界面框架与仪表盘静态显示
目标：创建一个车机风格的主界面（总大小800*480），包含四个部分：1.靠左侧的一列功能栏（80*470）包含关机键、音量+键、音量-键、空调设置键、应用菜单键。2.中间部分（420*470）用来放一张汽车图片，并且可以在图片上显示时速，油量等。3.右上角状态栏（280*150）显示日期、星期、时间、天气、地区等信息。4.右下角音乐播放状态栏（280*300）显示正在播放的音乐名称、进度条、暂停键、上一首键、下一首键
所需知识：
LVGL 对象创建（lv_obj_create）、样式设置（lv_style_set_*）。
lv_label 显示文字。
lv_timer 定时刷新时间。
验收：显示静态界面，时间每秒更新，按钮点击有响应。
🎵 第二阶段：音乐播放模块（集成 mplayer）
目标：两种启动方法：1.点击主界面右下角音乐播放状态栏的音乐播放键，默认播放第一首歌。2.点击左侧功能栏中的应用菜单键，进入应用菜单界面，选择“音乐播放器”进入音乐播放界面，在音乐列表中选择并播放。两种方法都支持暂停/继续、上一首/下一首。
所需知识：
文件扫描（opendir/readdir）获取 .mp3 文件列表。
fork + execlp 启动 mplayer slave 模式。
命名管道（FIFO）发送控制命令（pause、seek、volume）。
多线程读取 mplayer 输出（获取进度、时间、歌名）。
LVGL 列表控件（lv_list）、滑块（lv_slider）、按钮事件。
具体实现：
定义音乐播放界面函数 show_music_panel()，包含：播放列表、进度条、时间标签、音量滑块、控制按钮。
扫描 /mnt/sdcard/music下的 .mp3 文件，填充播放列表。
复用项目2中的 mplayer 控制代码（play_music、send_cmd、read_mplayer_output 线程）。
进度条更新：通过定时器读取全局变量 current_percent 并 lv_slider_set_value。
音量滑块事件：发送 volume <val> 1 命令。
验收：能播放音乐，进度条随播放移动，拖动滑块能调整音量，切换歌曲正常。
🎥 第三阶段：视频播放模块（可选，可复用项目2）
目标：点击左侧功能栏中的应用菜单键，进入应用菜单界面，选择“视频播放器”进入视频播放界面。视频播放器直接复用项目2
所需知识：同项目2，主要调用 mplayer 播放视频 
实现：基本与项目2类似
验收：能播放视频文件，画面显示正常，控制按钮可用。
🚦 第四阶段：数据模拟（CAN 模拟）
目标：模拟汽车 CAN 总线数据，动态更新中间部分的数据（车速、油量等）。
所需知识：
多线程：创建一个模拟数据线程（每 100ms 更新一次全局变量）。
互斥锁保护共享数据（pthread_mutex_t）。
验收：速度、油量有变化。
🌡️ 第五阶段：空调控制界面
目标：点击左侧栏中的空调设置键，显示空调调节界面（温度、风量、模式），支持滑块调节，数值实时显示（模拟）。
所需知识：
LVGL 滑块、标签、按钮样式。
滑动条事件回调，更新显示值。
具体实现：
创建空调面板，包含：温度调节滑块（16-32℃）、风量滑块（0-7）、模式按钮（制冷/制热/通风）。
滑块事件中更新旁边标签的数字。
点击“应用”按钮时，将设置值打印到串口或通过 socket 发送（暂只打印）。
验收：界面美观，滑块拖动时标签数字变化，点击应用打印正确值。
📡 第六阶段：网络通信集成（获取天气或导航信息）
目标：通过 TCP 连接局域网内的模拟服务器，接收天气预报（模拟数据）或导航方向指令，并在主界面顶部显示。
所需知识：
TCP socket 编程（socket, connect, recv）。
JSON 解析（推荐 cJSON 库）。
多线程：单独网络接收线程，接收后更新 UI（需用 lv_async_call 或定时器安全更新）。
具体实现：
在 PC 上用 Python 写一个简单 TCP 服务器，发送 JSON 格式消息：{"weather": "sunny", "temp": 26, "next_turn": "left"}。
开发板连接服务器 IP（可静态配置，如 192.168.137.100:8888）。
网络线程循环 recv，解析 JSON，存储到全局变量。
主界面定时器（每秒）读取天气/导航信息并刷新标签。
验收：开发板能收到服务器消息，并正确显示天气图标（用 LVGL 图标或文字）和导航提示。
🔧 第七阶段：设置与配置保存
目标：点击左侧功能栏中的应用菜单键，进入应用菜单界面，选择“设置”应用，进入设置界面，允许用户修改音量默认值、屏幕亮度、蓝牙名称等，并将设置保存到文件。
所需知识：
文件 I/O（fopen, fprintf, fscanf 或 JSON 文件）。
LVGL 开关（lv_switch）、下拉列表（lv_dropdown）、滑块。
具体实现：
创建设置界面，包含音量默认值（滑块）、亮度（滑块）、自动熄屏时间（下拉列表）。
在程序启动时读取 config.json 文件，加载设置。
修改后点击保存，将设置写入文件，并实时应用（如修改屏幕背光亮度通过 sysfs 写 /sys/class/backlight/.../brightness）。
验收：重启后设置仍保留。
🧩 第八阶段：集成与优化
目标：将所有模块集成到一个工程，使用主菜单导航，管理各界面生命周期，优化内存和 CPU 占用。
所需知识：
模块化编程（.h/.c 分离）。
界面切换时销毁旧对象（lv_obj_del），避免内存泄漏。
线程同步（退出时安全终止工作线程）。
具体实现：
创建主应用管理器，记录当前活动界面句柄。
实现 switch_to_music(), switch_to_ac(), switch_to_settings() 等函数，先销毁当前界面，再创建新界面。
统一处理网络线程、模拟数据线程的启动和停止（使用全局标志 app_running）。
确保所有 LVGL 对象在主线程中操作，使用 lv_async_call 跨线程更新 UI。
验收：所有功能无冲突，长时间运行稳定，界面切换流畅。
